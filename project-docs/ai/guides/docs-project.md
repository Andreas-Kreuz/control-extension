# Project Documentation Guide

## What to Update Alongside Code

When central concepts, contract terms, or project conventions change, check and update:

- Architecture docs: `project-docs/ARCHITECTURE.md` and per-component architectures (`ARCHITECTURE_LUA.md`, `ARCHITECTURE_SERVER.md`, `ARCHITECTURE_SHARED.md`, `ARCHITECTURE_APP.md`)
- READMEs: `README*.md` in relevant directories — user-facing overview and setup
- README_DEV files: developer-facing build/test/contribution notes

## Published Documentation

- Static docs / website: `pages/docs`
